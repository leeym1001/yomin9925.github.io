JVM 이란

JVM(Java Virtual Machine)은 java코드 또는 어플리케이션을 구동하기 위한 런타임 환경을 제공하는 엔진입니다. java 바이트 코드를 기계 언어로 변환합니다. JVM은 JRE(java Reuntime Environment)의 일부입니다. 다른 프로그래밍 언어에서 컴파일러는 특정 시스템에 대한 기계어 코드를 생성합니다. 그러나 java 컴파일러는 java 가상머신으로 알려진 가상 머신용 코드를 생성합니다.

JVM 아키텍처

  1. ClassLoader는 클래스 파일을 로드하는데 사용되는 서브시스템입니다. 3가지 주요기능으로 Loading, Linking,Initialization을 수행합니다.
  2. Method Area는 메타데이터, 상수 런타임 풀 및 메서드 코드와 같은 클래스 구조를 저장합니다.
  3. Heap은 모든 개체, 관련 인스턴스 변수 및 배열이 힙에 저장됩니다. 이 메모리는 공통이며 여러 스레드에서 공유됩니다.
  4. JVM language Stacks은 로컬 변수를 저장하며 일부의 결과입니다. 각 스레드에는 스레드가 생성될 때 동시에 생성되는 자체 JVM 스택이 있습니다. 메소드가 호출될 때마다 새로운 프레임이 생성되고 메소드 호출 프로세스가 완료되면 삭제됩니다.
  5. PC Registers는 현재 실행 중인 JVM(Java Virtual Machine) 명령의 주소를 저장합니다. Java에서 각 스레드에는 별도의 PC 레지스터가 있습니다.
  6. Native Method Stacks는 네이티브 라이브러리에 따라 네이티브 코드의 명령을 유지합니다. Java 대신 다른 언어로 작성되었습니다.
  7. Execution Engine은 java 바이트코드를 실행하기 위해서는 Execution Engine이 사용된다. 일반적으로 Java 바이트코드는 사람이 읽을 수 있는 형식으로 작성되기 때문입니다. 따라서 Execution Engine을 사용하면 사람이 읽을 수 있는 바이트 코드가 Java Virtual Machine의 기계에서 쉽게 실행할 수 있는 언어로 변환됩니다.
  8. Native Method interface는 프로그래밍 프레임워크입니다. JVM에서 실행 중인 Java 코드를 라이브러리 및 기본 애플리케이션에서 호출할 수 있습니다.
  9. Native Method Libraries는 Execution Engine에 필요한 Native Libraries(C, C++)의 모음입니다.

소프트웨어 코드 컴파일 및 실행 프로세스

소프트웨어 프로그램을 작성하고 실행하려면 다음이 필요합니다.

  1. Editor 프로그램을 입력하려면 메모장을 사용할 수 있습니다.
  2. Compiler 고급 언어 프로그램을 기본 기계 코드로 변환하기 위해 필요합니다.
  3. Linker 주 프로그램에서 서로 다른 프로그램의 파일을 참조하고 결합합니다.
  4. Loader 실행을 위해 하드 디스크, 플래시 드라이브, CD와 같은 보조 저장 장치에서 RAM으로 파일을 로드합니다. 코드를 실행하면 로딩이 자동으로 완료됩니다.
  5. Execution OS 및 프로세서에서 처리하는 코드의 실제 실행합니다.

C 코드 컴파일 및 실행 프로세스

java 컴파일 프로세스를 이해하기 전에 먼저 C에서 컴파일 및 링크 프로세스를 간단히 살펴보겠습니다.

  1. 메인에서 두 개의 함수 f1과 f2를 호출했다고 가정합니다. 메인 기능은 파일 a1.c에 저장됩니다. 그리고 각f함수는 a2.c, a3.c로 저장 후 컴파일에 소스파일 제공하면 출력값으로 해당 obj파일이 생성됩니다.
  2. 다음 단계는 링커의 도움을 받아 이러한 모든 obj 파일을 단일 .exe 파일로 통합하는 것입니다. 링커는 이러한 모든 파일을 함께 묶고 .exe 파일을 생성합니다.
  3. 프로그램이 실행되는 동안 로더 프로그램은 실행을 위해 a.exe를 RAM에 로드합니다. 아래 이미지는 전체 흐름을 보여준다.

Java VM에서 Java 코드 컴파일 및 실행

java에 대한 프로세스를 살펴보고, C컴파일과 비교해 보겠습니다.

  • 메인함수는 a1.java로 저장
  • f1함수는 a2.java로 저장
  • f2함수는 a3.java로 저장
  • 컴파일러는 3개의 파일을 컴파일하고 BYTE 코드로 구성된 3개의 해당 .class 파일을 생성합니다. C와 달리 링크가 수행되지 않습니다 .
  • Java VM 또는 Java Virtual Machine은 RAM에 상주합니다. 실행하는 동안 클래스 로더를 사용하여 클래스 파일을 RAM으로 가져옵니다(jvm architecture 이미지 참조). BYTE 코드는 모든 보안 위반에 대해 확인됩니다.
  • 다음으로 execution engine은 바이트코드를 네이티브 머신 코드로 변환합니다.

참고: JIT 또는 Just-in-time 컴파일러는 JVM(Java Virtual Machine)의 일부입니다. 유사한 기능을 가진 Byte Code의 일부를 동시에 해석합니다.
